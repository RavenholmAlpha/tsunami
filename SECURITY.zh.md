## 报告漏洞

如果你发现 TSUNAMI 中存在安全漏洞，请负责任地进行报告。

**请勿为安全漏洞创建公开的 GitHub Issue。**

### 如何报告

1. **邮件**：通过 GitHub 私密漏洞报告功能向项目维护者发送详细报告
2. **GitHub**：使用 [GitHub 的私密漏洞报告](https://github.com/RavenholmAlpha/tsunami/security/advisories/new)

### 报告内容

- 漏洞描述
- 复现步骤
- 潜在影响评估
- 建议修复方案（如有）

### 响应时间

- **确认**：48 小时内
- **初步评估**：7 天内
- **修复发布**：尽快，通常在 30 天内

### 范围

以下内容在安全报告范围内：

- 认证绕过或弱点
- TLS 配置漏洞
- 信息泄露
- 拒绝服务
- 权限提升
- 代码注入

### 范围外

- 社会工程攻击
- 物理攻击
- 依赖项中的问题（请直接向依赖项维护者报告）

## 安全最佳实践

在生产环境中部署 TSUNAMI 时：

1. **使用 Let's Encrypt** 或有效的 TLS 证书 — 避免在生产环境中使用 `--skip-verify`
2. **使用强密码** — 安装脚本默认生成 24 字节随机密码
3. **保持更新** — 定期运行 `tsunami-manage update`
4. **防火墙规则** — 仅暴露端口 443；将 SOCKS5/HTTP 代理绑定到 `127.0.0.1`
5. **监控日志** — `journalctl -u tsunami-server -f`
